# 使用说明

本文档说明本工具包包含的功能、如何运行、如何操作各功能，以及如何分享给同事使用。

---

## 一、工具包包含什么

本工具包含两个相对独立的功能模块：

| 功能 | 访问路径 | 说明 |
|------|----------|------|
| **4S 店对话分析与提示词改写** | 首页 `http://地址:5001/` | 上传对话 Excel，调用 Gemini 分析并改写 AI 电话机器人提示词 |
| **差额定率累进计算** | `http://地址:5001/progressive-rate` | 按档位与费率计算总费用，可保存/加载档位模板，可配置金额与费率单位 |
| **JSON 评分结果解析** | `http://地址:5001/json-eval` | 上传「id + JSON」Excel，解析为表格并导出 CSV |

---

## 二、环境与运行

### 2.1 本机运行（开发或临时使用）

**要求**：已安装 Python 3.8 及以上。

**步骤**：

1. 进入项目目录，安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
2. （可选）若使用「对话分析 / 提示词改写」或「JSON 解析」中依赖 Gemini 的部分，需配置 API Key：
   ```bash
   cp .env.example .env
   # 编辑 .env，填入 GEMINI_API_KEY
   ```
   **仅使用「差额定率累进计算」无需配置 .env。**
3. 启动服务：
   ```bash
   python app.py
   ```
4. 浏览器访问：
   - 首页：http://127.0.0.1:5001
   - 差额定率计算：http://127.0.0.1:5001/progressive-rate
   - JSON 解析：http://127.0.0.1:5001/json-eval

### 2.2 独立 HTML 文件（无需 Python 与服务器）

项目内提供 **`progressive_rate_standalone.html`**，为差额定率累进计算的单文件版本：

- **直接双击用浏览器打开**即可使用（地址栏为 `file:///...`），无需安装 Python、无需启动任何服务。
- **放到任意网页服务器**（如公司内网站点、Nginx、GitHub Pages 等）后，同事通过该页面的 URL 即可访问，功能与 Flask 版一致（档位模板保存在浏览器 localStorage）。
- 以 `file://` 打开时页头「复制链接」会提示“当前为本地文件，无链接可复制”，需将本文件部署到可访问的网址后再复制链接分享。

适合：只使用差额定率计算、且希望用单个文件分发或挂到现有网站的场合。

### 2.3 服务器部署（同事通过链接直接使用，Flask 全功能）

若希望同事在自己电脑上打开链接即可使用**整套工具**（含对话分析、JSON 解析等），需将本项目部署到一台大家都能访问的服务器（公司内网或云服务器）。详细步骤见 **DEPLOY.md**。

部署完成后，将链接发给同事即可，例如：
- 差额定率计算：`http://服务器IP或域名:5001/progressive-rate`
- 或使用 Nginx 时：`http://域名/progressive-rate`

## 三、功能一：4S 店对话分析与提示词改写

- **入口**：浏览器打开 `http://地址:5001/`
- **用途**：上传店内真实对话数据（Excel），由 AI 分析流程与话术，并基于分析结果改写你提供的 AI 电话机器人提示词。
- **前置**：需在 `.env` 中配置 `GEMINI_API_KEY`（在 [Google AI Studio](https://aistudio.google.com/app/apikey) 获取）。
- **简要流程**：
  1. 填写或确认 Gemini API Key（页面上或 .env）。
  2. 上传对话数据 Excel（须含「对话ID」「对话内容」列）。
  3. 可选：修改 System Instructions（留空则用默认）。
  4. 输入或粘贴原有机器人提示词。
  5. 点击「执行对话分析&提示词改写」，等待完成后在结果区查看/复制改写后的提示词。

更细的 Excel 格式与操作说明见项目 **README.md**。

## 四、功能二：差额定率累进计算（重点说明）

- **入口**：浏览器打开 `http://地址:5001/progressive-rate`
- **用途**：根据「计费基数」和自定义「档位 + 费率」，按差额定率累进规则计算总费用；可保存/加载档位方案，便于复用和分享页面链接给同事使用。

### 4.1 页面结构

- **页头**：标题、简要说明、分享提示，以及「当前页链接 + 复制链接」按钮。
- **左侧栏「常用档位模板」**：
  - **新建模板**：清空当前档位，从空白开始配置。
  - **已保存模板列表**：每个模板显示名称；**编辑**加载到右侧使用，**重命名**/**删除**可修改或删除该模板。
- **右侧主区域**：
  - **计费基数与总费用**：输入计费基数，点击「计算总费用」，下方显示总费用（万元或你选的金额单位）。
  - **档位配置**：金额单位、费率单位、档位表格（区间下限、区间上限、费率），以及「保存方案」、每档的「+」「×」操作。

### 4.2 单位配置（在「档位配置」卡片内）

- **金额单位**：元 / 万元 / 亿元，仅影响界面显示。
- **费率单位**：‰（千分比）或 %（百分比），会参与计算（内部按千分或百分换算）。

### 4.3 档位配置与计算规则

- **档位**：每行一个档位，包含「区间下限」「区间上限」「费率」。
  - 区间上限留空表示该档为「以上」无上限。
  - 最后一档右侧有 **+** 和 **×**：**+** 在本档下方新增一行，**×** 删除本档；非最后一档仅显示 **×**。
- **计算逻辑**：总费用 = 各档「落在该档内的基数段 × 该档费率」之和。  
  例如基数 100000（万元），档位 0–200、200–500、…、50000 以上，则 0–200 段按第 1 档费率算，200–500 段按第 2 档费率算，以此类推，最后一段为 50000–100000 按最后一档费率算，全部相加即总费用。

### 4.4 保存与加载方案

- **保存方案**：在档位配置区点击「保存方案」，输入方案名称后确定，当前档位会保存到左侧「常用档位模板」列表（仅存在当前浏览器）。
- **加载方案**：在左侧列表点击某模板的「编辑」，将该模板的档位加载到右侧，即可修改或直接用于计算。
- **新建模板**：点击左侧「新建模板」可清空当前档位，重新配置后如需保留可再「保存方案」。

### 4.5 分享给同事

- 页头会显示当前页面完整链接；点击「复制链接」可复制到剪贴板。
- 若部署在服务器上，将此链接发给同事，同事用浏览器打开即可使用，无需安装任何软件。
- 若在本机运行，链接中的 `localhost` 需改为本机 IP（如 `192.168.1.100:5001`），且同事需与你在同一局域网才能访问。

## 五、功能三：JSON 评分结果解析

- **入口**：`http://地址:5001/json-eval`
- **用途**：上传「第一列为 id，后面列为 JSON 内容」的 Excel，解析为结构化表格，并可导出 CSV。
- **操作**：按页面提示上传符合格式的 Excel，查看解析结果后点击「导出为 CSV」即可。

## 六、常见问题

| 问题 | 处理建议 |
|------|----------|
| 差额定率页打不开 | 确认服务已启动（`python app.py` 或按 DEPLOY.md 部署），且访问的地址和端口正确（默认 5001）。 |
| 同事打不开我发的链接 | 若你本机运行：确认同事与你在同一网络，且链接中为你的本机 IP 而非 localhost。若部署在服务器：确认防火墙放行 5001（或 Nginx 所用端口）。 |
| 保存的档位模板不见了 | 模板存在浏览器本地（localStorage），清除站点数据或换浏览器/电脑会丢失。重要方案请自行备份（如截图或记录档位）。 |
| 对话分析/改写报错 | 检查 .env 中 GEMINI_API_KEY 是否有效、网络是否能访问 Google API；可尝试更换模型（见 README 中 GEMINI_MODEL 说明）。 |
