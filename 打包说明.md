# 打包说明

本文说明如何将本工具「打包」交付给他人（同事、客户或部署到服务器），以及交付物应包含哪些内容。

---

## 一、交付物清单（建议包含）

打包时请包含以下文件与目录，保证对方能完整运行和查阅文档：

```
conversation-analysis-tool/
├── app.py                 # 主程序入口
├── gemini_rest.py         # Gemini API 调用
├── requirements.txt      # Python 依赖
├── .env.example           # 环境变量示例（复制为 .env 使用）
├── README.md              # 项目说明与对话分析说明
├── 使用说明.md             # 完整使用说明（本工具包使用指南）
├── DEPLOY.md              # 服务器部署指南
├── 打包说明.md             # 本文件
├── progressive_rate_standalone.html  # 差额定率计算独立单页（可单独分发，无需 Python）
├── templates/             # 页面模板
│   ├── index.html
│   ├── progressive_rate.html
│   └── json_eval.html
├── static/                # 静态资源
│   ├── style.css
│   ├── app.js
│   └── json_eval.js
└── （可选）sample_data.py  # 示例脚本，非运行必需
```

**不要打包**（若存在）：

- `venv/`、`__pycache__/`、`.env`（含密钥）、`*.pyc`、`.git/` 等，避免体积过大或泄露配置。

---

## 二、打包方式

### 方式一：压缩为 zip（推荐）

在**上一级目录**执行（避免把父目录名也压进去）：

```bash
# 进入项目上一级目录后执行，生成 conversation-analysis-tool.zip
zip -r conversation-analysis-tool.zip conversation-analysis-tool \
  -x "conversation-analysis-tool/venv/*" \
  -x "conversation-analysis-tool/__pycache__/*" \
  -x "conversation-analysis-tool/.env" \
  -x "conversation-analysis-tool/.git/*"
```

Windows 下可用 7-Zip、WinRAR 或资源管理器右键「压缩」项目文件夹，并排除上述目录和 `.env`。

### 方式二：Git 仓库

若对方使用 Git，可直接提供仓库地址或打包为不含 `.git` 的压缩包：

```bash
git clone <仓库地址>
# 或
git archive -o conversation-analysis-tool.zip HEAD
```

## 三、交付时一并说明

建议在交付压缩包或链接时，附上简短说明，例如：

1. **运行方式**：解压后执行 `pip install -r requirements.txt` 和 `python app.py`，浏览器访问 `http://127.0.0.1:5001`；差额定率页为 `http://127.0.0.1:5001/progressive-rate`。
2. **使用文档**：解压后请先阅读 **使用说明.md**，按需配置 `.env`（仅对话分析/改写需要）。
3. **服务器部署**：若需多人通过链接访问，请按 **DEPLOY.md** 部署到服务器后再分享链接。

---

## 四、版本与更新

- 交付时可在 `README.md` 或 `使用说明.md` 开头注明版本号或日期，便于后续区分。
- 更新后再打包时，建议排除与本次相同的无关文件，并更新使用说明中的变更说明。
